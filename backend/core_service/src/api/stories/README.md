# Stories API

## Authentication

Для защиты ресурсов используется JWT-токен, который передается в заголовке `Authorization`:

```
Authorization: Bearer <your_jwt_token>
```

## Роли пользователей

| Роль | Описание |
|------|----------|
| `user` | Обычный пользователь (может создавать истории и просматривать их) |
| `admin` | Администратор (имеет доступ к управлению статусами историй и тегами) |

---

## Endpoints

### 1. Get All Tags

**GET** `/stories/tags`

Краткое описание: Получение списка всех доступных тегов для историй.

#### Параметры

Этот endpoint не требует аутентификации.

#### Пример запроса

```
GET /stories/tags
```

#### Пример ответа (успех)

```json
{
  "tags": [
    {
      "id": 1,
      "name": "python"
    },
    {
      "id": 2,
      "name": "javascript"
    },
    {
      "id": 3,
      "name": "database"
    }
  ]
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `500 Internal Server Error` | Внутренняя ошибка сервера |

---

### 2. Create Tag (только администратор)

**POST** `/stories/new_tag`

Краткое описание: Создание нового тега для историй. Доступно только администраторам.

#### Параметры

**Заголовки**:

| Заголовок | Обязательный | Описание |
|-----------|--------------|----------|
| `Authorization` | Да | JWT-токен в формате `Bearer <token>`, где токен принадлежит пользователю с ролью `admin` |

**Тело запроса (Body)**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `name` | string | Да | Название нового тега |

#### Пример запроса

```json
{
  "name": "backend"
}
```

#### Пример ответа (успех)

```json
{
  "created": {
    "id": 4,
    "name": "backend"
  }
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `400 Bad Request` | Неверный формат токена |
| `403 Forbidden` | Пользователь не является администратором |
| `404 Not Found` | Пользователь не найден (токер содержит несуществующий ID) |
| `500 Internal Server Error` | Внутренняя ошибка сервера |

---

### 3. Get Stories Requests (только администратор)

**GET** `/stories/requests`

Краткое описание: Получение списка запросов на публикацию историй. Доступно только администраторам.

#### Параметры

**Query параметры**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `status` | string | Нет | Статус истории: `new`, `approved`, `rejected` (по умолчанию: `new`) |
| `limit` | integer | Нет | Количество записей на странице (по умолчанию: 5) |
| `offset` | integer | Нет | Смещение для пагинации (по умолчанию: 0) |

**Заголовки**:

| Заголовок | Обязательный | Описание |
|-----------|--------------|----------|
| `Authorization` | Да | JWT-токен в формате `Bearer <token>`, где токен принадлежит пользователю с ролью `admin` |

#### Пример запроса

```
GET /stories/requests?status=new&limit=10&offset=0
```

#### Пример ответа (успех)

```json
{
  "stories": [
    {
      "id": 1,
      "author_id": "123e4567-e89b-12d3-a456-426614174000",
      "title": "Как я изучал Python",
      "content": "В этой статье я расскажу...",
      "status": "new",
      "created_at": "2026-06-22T10:30:00Z",
      "updated_at": "2026-06-22T10:30:00Z",
      "tags": ["python", "education"],
      "images": ["abc123.jpg", "def456.jpg"]
    },
    {
      "id": 2,
      "author_id": "123e4567-e89b-12d3-a456-426614174001",
      "title": "Оптимизация баз данных",
      "content": "Оптимизация — важная часть разработки...",
      "status": "new",
      "created_at": "2026-06-21T09:15:00Z",
      "updated_at": "2026-06-21T09:15:00Z",
      "tags": ["database", "performance"],
      "images": ["xyz789.jpg"]
    }
  ]
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `400 Bad Request` | Неверный формат токена или неверный статус |
| `403 Forbidden` | Пользователь не является администратором |
| `404 Not Found` | Пользователь не найден (токер содержит несуществующий ID) |
| `500 Internal Server Error` | Внутренняя ошибка сервера |

### 4. Create Story

**POST** `/stories/new`

Краткое описание: Создание новой истории. Доступно всем авторизованным пользователям.

#### Параметры

**Заголовки**:

| Заголовок | Обязательный | Описание |
|-----------|--------------|----------|
| `Authorization` | Да | JWT-токен в формате `Bearer <token>` |

**Тело запроса (Body)**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `title` | string | Да | Заголовок истории |
| `tags` | array of integers | Да | Список ID тегов |
| `content` | string | Да | Текстовое содержимое истории |
| `images` | array of strings | Да | Список изображений в Base64 |

> **Примечание**: Изображения сохраняются асинхронно в фоновом режиме.

#### Пример запроса

```json
{
  "title": "Мой первый опыт с FastAPI",
  "tags": [1, 3],
  "content": "FastAPI — это современный веб-фреймворк для Python...",
  "images": [
    "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQE...",
    "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..."
  ]
}
```

#### Пример ответа (успех)

```json
{
  "status": "Story Created"
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `400 Bad Request` | Неверный формат токена или неверный формат изображения |
| `401 Unauthorized` | Пользователь не найден (токер содержит несуществующий ID) |
| `500 Internal Server Error` | Внутренняя ошибка сервера |

---

### 5. Change Story Status (только администратор)

**PATCH** `/stories/status`

Краткое описание: Изменение статуса истории (одобрить/отклонить). Доступно только администраторам.

#### Параметры

**Заголовки**:

| Заголовок | Обязательный | Описание |
|-----------|--------------|----------|
| `Authorization` | Да | JWT-токен в формате `Bearer <token>`, где токен принадлежит пользователю с ролью `admin` |

**Тело запроса (Body)**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `story_id` | integer | Да | Идентификатор истории |
| `new_status` | string | Да | Новый статус: `new`, `approved`, `rejected` |

#### Пример запроса

```json
{
  "story_id": 1,
  "new_status": "approved"
}
```

#### Пример ответа (успех)

```json
{
  "updated": true
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `400 Bad Request` | Неверный формат токена или неверный статус |
| `403 Forbidden` | Пользователь не является администратором |
| `404 Not Found` | Пользователь или история не найдена |
| `500 Internal Server Error` | Внутренняя ошибка сервера |

## Статусы историй

Для управления жизненным циклом историй используются следующие статусы:

| Значение | Описание |
|----------|----------|
| `new` | Новая история, ожидает рассмотрения |
| `approved` | История одобрена администратором |
| `rejected` | История отклонена администратором |

---

## Пагинация

По умолчанию используется:
- `limit` = 5 (количество записей на странице)
- `offset` = 0 (смещение)

Пример:
```
GET /stories/requests?status=approved&limit=10&offset=20
```

---

## Формат изображений

Изображения передаются в формате Base64 с префиксом MIME-типа:

```
data:image/jpeg;base64,/9j/4AAQSkZJRgABAQE...
data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...
```

Поддерживаемые форматы: JPEG, PNG (и другие, определяемые MIME-типом).

---

## Примечания

1. Stories с статусом `approved` и `rejected` доступны для просмотра через `/stories/requests`.
2. Истории с статусом `new` создаются при публикации и ожидают одобрения администратора.
3. Изображения сохраняются асинхронно в фоновом режиме в директорию, указанную в `cfg_obj.UPLOAD_DIR`.
4. При создании истории теги должны существовать в базе данных (иначе они будут проигнорированы).
5. Пользователь может создавать истории, но не может их редактировать или удалять.
6. Администратор может просматривать все истории и управлять их статусами.
7. Все ответы с историями содержат полную информацию, включая теги и изображения.
