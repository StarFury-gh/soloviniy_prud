# Events API

## Authentication

Для защиты ресурсов используется JWT-токен, который передается в заголовке `Authorization`:

```
Authorization: Bearer <your_jwt_token>
```

## Роли пользователей

| Роль | Описание |
|------|----------|
| `user` | Обычный пользователь |
| `admin` | Администратор (имеет доступ к созданию, редактированию и удалению событий) |

---

## Endpoints

### 1. Create Event (только администратор)

**POST** `/events/new`

Краткое описание: Создание нового события. Доступно только администраторам.

#### Параметры

**Заголовки**:

| Заголовок | Обязательный | Описание |
|-----------|--------------|----------|
| `Authorization` | Да | JWT-токен в формате `Bearer <token>`, где токен принадлежит пользователю с ролью `admin` |

**Тело запроса (Body)**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `name` | string | Да | Название события |
| `description` | string | Да | Описание события |
| `date` | datetime (ISO 8601) | Да | Дата и время события |
| `images` | array of strings | Нет | Список изображений в Base64 (по умолчанию: `[]`) |

#### Пример запроса

```json
{
  "name": "Встреча сообщества",
  "description": "Ежемесячная встреча разработчиков",
  "date": "2026-07-15T18:00:00Z",
  "images": [
    "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQE...",
    "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..."
  ]
}
```

#### Пример ответа (успех)

```json
{
  "created": {
    "id": 1,
    "name": "Встреча сообщества",
    "description": "Ежемесячная встреча разработчиков",
    "created_at": "2026-06-22T10:30:00Z",
    "updated_at": "2026-06-22T10:30:00Z",
    "created_by": "123e4567-e89b-12d3-a456-426614174000",
    "date": "2026-07-15T18:00:00Z",
    "images": []
  }
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `400 Bad Request` | Неверный формат токена |
| `403 Forbidden` | Пользователь не является администратором |
| `404 Not Found` | Пользователь не найден (токер содержит несуществующий ID) |
| `500 Internal Server Error` | Внутренняя ошибка сервера |

### 2. Get Events

**GET** `/events/`

Краткое описание: Получение списка событий с пагинацией.

#### Параметры

**Query параметры**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `event_status` | string | Да | Статус события: `past` (прошедшие) или `future` (будущие) |
| `limit` | integer | Нет | Количество записей на странице (по умолчанию: 5) |
| `offset` | integer | Нет | Смещение для пагинации (по умолчанию: 0) |

**Заголовки**:

| Заголовок | Значение |
|-----------|----------|
| `Authorization` | `Bearer <token>` (опционально) |

#### Пример запроса

```
GET /events/?event_status=future&limit=10&offset=0
```

#### Пример ответа (успех)

```json
{

    {

      "updated_at": "2026-06-22T10:30:00Z",
      "created_by": "123e4567-e89b-12d3-a456-426614174000",
      "date": "2026-07-15T18:00:00Z",
      "images": []
    },
    {
      "id": 2,
      "name": "Мастер-класс по Python",
      "description": "Глубокое погружение в асинхронное программирование",
      "created_at": "2026-06-21T09:15:00Z",
      "updated_at": "2026-06-21T09:15:00Z",
      "created_by": "123e4567-e89b-12d3-a456-426614174001",
      "date": "2026-08-20T14:00:00Z",
      "images": []
    }
  ]
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `400 Bad Request` | Неверный `event_status` (должен быть `past` или `future`) |

### 3. Update Event (только администратор)

**PATCH** `/events/{event_id}`

Краткое описание: Редактирование существующего события. Доступно только администраторам.

#### Параметры

**Путь (Path)**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `event_id` | integer | Да | Идентификатор события |

**Заголовки**:

| Заголовок | Обязательный | Описание |
|-----------|--------------|----------|
| `Authorization` | Да | JWT-токен в формате `Bearer <token>`, где токен принадлежит пользователю с ролью `admin` |

**Тело запроса (Body)**:

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `new_name` | string | Нет | Новое название события |
| `new_description` | string | Нет | Новое описание события |
| `new_date` | datetime (ISO 8601) | Нет | Новая дата и время события |

> **Примечание**: Хотя бы одно поле должно быть указано для обновления.

#### Пример запроса

```json
{
  "new_name": "Обновленная встреча сообщества",
  "new_description": "Ежемесячная встреча разработчиков с обновленной программой"
}
```

#### Пример ответа (успех)

```json
{
  "updated": {
    "id": 1,
    "name": "Обновленная встреча сообщества",
    "description": "Ежемесячная встреча разработчиков с обновленной программой",
    "created_at": "2026-06-22T10:30:00Z",
    "updated_at": "2026-06-22T12:00:00Z",
    "created_by": "123e4567-e89b-12d3-a456-426614174000",
    "date": "2026-07-15T18:00:00Z",
    "images": []
  }
}
```

#### Ошибки

| Код | Причина |
|-----|---------|
| `400 Bad Request` | Неверный формат токена или отсутствие полей для обновления |
| `403 Forbidden` | Пользователь не является администратором |
| `404 Not Found` | Событие с указанным `event_id` не найдено |
| `500 Internal Server Error` | Внутренняя ошибка сервера |

## Описание полей ответов

### Event Object

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | integer | Уникальный идентификатор события |
| `name` | string | Название события |
| `description` | string | Описание события |
| `created_at` | datetime (ISO 8601) | Дата и время создания события |
| `updated_at` | datetime (ISO 8601) | Дата и время последнего обновления |
| `created_by` | string (UUID) | ID пользователя, создавшего событие |
| `date` | datetime (ISO 8601) | Дата и время самого события |
| `images` | array of strings | Список путей к изображениям |

---

## Статусы событий

Для фильтрации событий используются следующие статусы:

| Значение | Описание |
|----------|----------|
| `past` | Прошедшие события (дата < текущая дата) |
| `future` | Будущие события (дата >= текущая дата) |

---

## Пагинация

По умолчанию используется:
- `limit` = 5 (количество записей на странице)
- `offset` = 0 (смещение)

Пример:
```
GET /events/?event_status=future&limit=10&offset=20
```

---

## Примечания

1. Все ресурсы, кроме `/events/`, требуют аутентификации администратора.
2. Дата и время должны быть в формате ISO 8601 (например: `2026-07-15T18:00:00Z`).
3. Изображения передаются в формате Base64 при создании события.
4. При обновлении события достаточно указать только те поля, которые нужно изменить.
5. Событие считается прошедшим, если его дата меньше текущей даты и времени (UTC).
6. При обновлении события поле `updated_at` автоматически обновляется на текущее время.
